Dispatch Center

Descripción.

Este proyecto tiene como objetivo la administración eficiente de envíos y recolectas, la creación de rutas optimizadas para mensajeros, la administración de negocios y mensajeros, la definición de zonas en un mapa para gestionar envíos, el historial de tareas, y la implementación de una estación de sorting para empaquetar pedidos.

Metodologia empleada: Design Thinking y Lean UX

1. Empatizar

Nuestros usuarios.

Al ser una herramienta espacial para la administración interna nos enfocamos a entender las necesidades del negocio y en observar el proceso actual de envíos y recolectas para identificar puntos débiles y áreas de mejora.

Hallazgos
  1. La acción prinicpal del dispatch es asignar envíos y recolectas a los mensajeros.
  2. Es importante desarrollar un sistema de notificaciones para mantener a los usuarios informados.
  3. Considerar un algoritmo que permita optimizar las rutas y minimizar el tiempo de entrega.
  4. División del mapa en zonas para gestionar de manera eficiente las áreas de envío y los mensajeros.
  5. Es importante adecuar la cantidad de paquetes a entregar dependiendo del vehiculo del mensajero (bicicleta o carro).
  6. Registro completo de todas las tareas realizadas con posibilidad de filtrar y buscar tareas específicas.
  7. Herramientas para optimizar el proceso de sorting que es el área designada para clasificar y entregar paquetes.
  8. Administración de los perfiles de negocios y mensajeros.
  9. Considerar una sección de Analytics para la toma interna de desiciones interna.
2. Definir

Análisis de Datos.

De acuerdo al analisis realizado definimos el siguiente problema principal a resolver, el objetivo es obtener un MVP y mejorar el producto de forma continua basános en el uso y feedback continuo de los usuarios.

Los patrones comunes encontrados son los siguientes:

  • El 80% de los mensajeros encuestados mencionan que una mejor planificación de rutas mejoraría significativamente su eficiencia y rapidez en entrgar pedidos.
  • El no tener un sistema y proceso interno adecuado hace que las rutas no considen el tráfico en tiempo real y el soporte a mensajeros sea lento.
  • Es importante para la operación tener informacion de las tareas en tiempo real por ejemplo: Negocio A tiene 10 pedidos pendientes y 5 recolectas programadas para el día
Problema central

Diseñar un dashboard que permita visualizar todos los envíos y recolecciones de manera centralizada, proporcionando a los usuarios una vista completa y detallada de todas las operaciones. Este dashboard incluirá un algoritmo avanzado que optimice las rutas de entrega, asegurando así la máxima eficiencia en tiempo y recursos. Además, se integrarán funciones adicionales para el seguimiento en tiempo real de los envíos, la generación de reportes personalizados y alertas automáticas para cualquier incidencia o retraso.

5. Testear

Validamos nuestras ideas.

Realizamos pruebas de usabilidad con el dispatcher a cargo en bodega, pidiéndole que completará tareas específicas a continuación los resultados:

  1. Las tareas se completaron en el prototipo satisfactoriamente.
  2. El tiempo para asignar una ruta fué de 3 minutos reduciendo un 37% el tiempo original de 8 minutos.
Conclusiones

Este proyecto de dispatch de envíos tiene como objetivo mejorar la eficiencia y la experiencia del usuario mediante la implementación de una metodología de Design Thinking y Lean UX. La clave del éxito radica en la iteración continua y la adaptabilidad a las necesidades cambiantes de los usuarios.
